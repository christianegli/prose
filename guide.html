A two-minute guide to GitHub sync and annotations

1 — Connect to GitHub

1

Open Settings

Press ⌘ , (or use the Command Palette ⌘ K → "Settings"). The settings panel slides open.

2

Enter your GitHub credentials

You need a GitHub Personal Access Token with repo scope. Create one at github.com/settings/tokens/new.

3

Sync your document

⌘ ⇧ S pushes your document to GitHub. The dot in the bottom-right corner shows the sync state:

Synced
Local changes (not yet pushed)
Error (check token/repo)

To pull the latest version from GitHub: ⌘ ⇧ P

2 — Annotations

1

Select text and annotate

Highlight any text in your document, then press ⌘ ⇧ A. The annotation modal appears.

The problem with comfort is not that it feels good — it is that it quietly removes the need to think.
2

Write your note

Type your annotation in the modal. You can assign it to Atlas for review — Atlas will receive it via Telegram and can respond with feedback.

3

The highlight appears inline

Annotated text gets a subtle highlight. Yellow = your note. Green = assigned to Atlas. Hover to see the note in a popover.

4

Atlas responds via Telegram

When you send an annotation to Atlas, it arrives as a Telegram message with the quoted text and your note. Atlas can reply directly.

5

Review all annotations

Press ⌘ ⇧ I to open the annotations panel. All notes for the current document are listed. You can delete or send any annotation to Atlas from here.
